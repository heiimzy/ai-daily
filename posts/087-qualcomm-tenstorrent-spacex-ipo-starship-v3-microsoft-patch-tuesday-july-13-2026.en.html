⚡ July 13 Tech Roundup: Qualcomm Eyes $8-10B Tenstorrent Acquisition, SpaceX Files $75B IPO with Starship V3 Success, Microsoft Patches Record 200+ Vulnerabilities

1. Qualcomm and Tenstorrent: A New Contender in the AI Chip Arena

Qualcomm is in early-stage talks to acquire Tenstorrent, the AI chip company founded by legendary chip architect Jim Keller, for an estimated $8-10 billion, multiple outlets reported July 12. If finalized, it would be the most significant AI chip M&A since Intel's $5.4 billion acquisition of Habana Labs in 2019.

Tenstorrent designs AI chips using the open RISC-V instruction set architecture, avoiding Arm and x86 licensing fees. Jim Keller — who previously led AMD's Zen CPU revival, Apple's A4/A5 chips, Tesla's FSD chip, and Intel's chip programs — brings unparalleled engineering credibility. An $8-10 billion valuation for a pre-revenue-at-scale company reflects a bet on that engineering pedigree rather than current cash flows.

Strategically, the deal makes perfect sense for Qualcomm: the company dominates edge AI through Snapdragon but has no competitive data center AI accelerator. Tenstorrent's RISC-V architecture is explicitly designed to compete with Nvidia's H-series GPUs. The talks, still early and unconfirmed, would give Qualcomm real seats at the AI hardware table.

"The $8-10 billion valuation is more of a bet on Keller's engineering team than on Tenstorrent's current revenue." — Industry analyst

2. SpaceX Files $75B IPO, Starship V3 Completes Maiden Flight

SpaceX has officially filed its IPO prospectus with the SEC, seeking to raise up to $75 billion through a common stock offering at an estimated $1.8 trillion valuation. The filing represents not just a capital raise but a strategic vision spanning space launch, Starlink satellite internet, and space-based AI computing.

Starship V3 First Flight

On Friday July 10, SpaceX launched its 12th Starship test flight from Texas — the first for both the V3 version and a new launch pad. Despite one upper-stage engine failure, Starship successfully reached suborbital cruise and splashed down in the Indian Ocean about an hour later. The V3 upgrade significantly improves payload capacity (100-200 tons to orbit) and reusability, aiming to dramatically reduce launch costs.

IFT-13 Set for July 16

SpaceX has announced that Starship's 13th test flight (IFT-13) is targeting July 16 with a 90-minute launch window opening at 5:45 PM CT. For the first time, the mission will carry 20 next-generation Starlink V3 satellites, each with 1 Tbps capacity, marking a major step in scaling the satellite internet constellation.

The prospectus also reveals SpaceX's AI infrastructure ambitions: the acquisition of xAI (completed February 2026), a $1.25 billion monthly compute lease agreement with Anthropic, and plans to deploy orbital AI computing satellites from 2028. Proceeds will fund Starship infrastructure, Starlink V3 deployment, and the Colossus supercomputing cluster.

3. Microsoft's Record Patch Tuesday: AI-Driven Vulnerability Discovery Reshapes Security

Microsoft's July Patch Tuesday fixed over 200 security vulnerabilities — a record for a single Patch Tuesday release — reflecting the profound impact of AI on vulnerability discovery. Tom Gallagher, VP of engineering at Microsoft's Security Response Center, acknowledged that AI tools are fueling a surge in vulnerability discovery and predicted that Patch Tuesday releases will continue to grow.

Most Critical: CVE-2026-45657

Rated 9.8/10 severity, this wormable Windows core vulnerability allows remote attackers to take full control of a machine with zero user interaction — reminiscent of the 2017 WannaCry attack. Researchers urge organizations to patch immediately despite Microsoft's "less likely to be exploited" rating.

CVE-2026-41091 (7.8/10), already exploited in the wild, affects Microsoft Defender and allows attackers with a foothold to gain full system control. Three zero-day flaws were disclosed including a BitLocker bypass that could expose stolen Windows laptops.

The AI Security Paradox

A researcher known as "Nightmare Eclipse" has been locked in a standoff with Microsoft for months, posting exploit code for unpatched Windows flaws on GitHub since April. The researcher has threatened to release more exploit code on July 14 — the next Patch Tuesday. The UK's NCSC had warned in April of a wave of urgent updates driven by AI-assisted discovery, a prediction now materializing.

4. Grok Build CLI Security Controversy: Entire Repos Uploaded to xAI Servers

A security researcher published a wire-level teardown of xAI's Grok Build CLI, revealing that the tool uploads users' complete code repositories to xAI's Google Cloud Storage bucket — including files the agent never read, touched, or was asked to share.

Testing on a 12GB repository showed 5.1GB successfully transferred, including unredacted credentials from .env files. Even after disabling the "Improve the model" toggle, code uploads continued uninterrupted. xAI had marketed Grok Build as "local-first," claiming "nothing from your codebase is transmitted to xAI servers during a session." The wire-level data directly contradicts this.

"Grok Build runs two data channels simultaneously. The first is the model-turn channel: files the agent reads sent as context. The second is a background upload of the entire workspace as a git bundle — sent via a separate POST /v1/storage endpoint to a bucket called grok-code-session-traces."

The practical response for teams using Grok Build: treat the CLI as a cloud service that may upload more than the files the agent touches. Run it only in repositories you can send to xAI, keep real credentials out of tracked files, and test network behavior before allowing it in regulated environments.

As AI infrastructure investment continues surging through 2026, these trends will shape the technology landscape for the next 12 months.
