⚡ July 16 Tech Roundup: xAI Open-Sources Grok Build Amid Privacy Fallout, GPT-5.6 Sol Pens 50-Year Math Proof in Under an Hour, NVIDIA NemoClaw Matches Top Closed Model at 1/10 Cost

1. xAI Open-Sources Grok Build After Privacy Scandal

On July 16 (Japan time), SpaceXAI (formerly xAI) officially open-sourced Grok Build, its AI coding agent framework, on GitHub under a permissive license. The company also reset usage limits for all users, marking xAI's strategic shift from model-layer competition to developer tooling and ecosystem building.

Originally launched in May 2026 as a beta, Grok Build leverages the Grok 4.5 model for a full terminal-based interactive coding experience. It autonomously executes coding tasks, supports Plan mode for coding blueprints, and extends functionality via Skills and MCP servers. Its full-screen TUI, combined with mouse interaction support, breaks traditional terminal conventions by enabling click-and-scroll agent management.

Privacy Controversy: Entire Codebases Uploaded to GCS

The open-sourcing comes on the heels of a serious privacy scandal. Security researcher cereblab published a wire-level analysis of Grok Build CLI v0.2.93 on July 10, revealing:

  • Dual-channel data transmission: Beyond the normal model inference channel, Grok Build operated a second channel that packaged the entire Git repository as a bundle and uploaded it to xAI's Google Cloud Storage bucket grok-code-session-traces via POST /v1/storage
  • 27,800× data volume ratio: On a 12 GB test repository, the model channel transmitted just 192 KB while the storage channel moved 5.10 GB
  • Privacy toggle ineffective: Disabling "Improve the model" had no effect — the server continued returning trace_upload_enabled: true
  • Credentials in cleartext: .env files with API keys were transmitted unredacted in request bodies

The report hit the front page of Hacker News within days. xAI disabled the upload server-side on July 12 via the disable_codebase_upload: true flag, though the upload code remains in the binary. Elon Musk promised "complete and utter deletion" of all previously uploaded user data. On July 15, xAI confirmed it had disabled data upload by default and deleted all stored coding data. With the code now open-source, the community can independently audit the tool's behavior.

2. GPT-5.6 Sol Ultra Claims Proof of 50-Year-Old Math Conjecture in Under an Hour

OpenAI's new flagship model GPT-5.6 Sol Ultra has claimed a dramatic breakthrough — producing a full proof of the Cycle Double Cover Conjecture, an open problem in graph theory dating back to the 1970s, in under one hour. However, the mathematics community has not yet verified the proof.

How It Worked: 64 Parallel Agents

OpenAI published the complete prompt and proof documentation on July 10. Sol Ultra was configured to run up to 64 reasoning agents in parallel with a sophisticated discovery and verification mechanism:

  • Diverse exploration: Early rounds kept agent approaches diverse to avoid premature convergence on local optima
  • Attacker agents: A dedicated team of "attacker" agents scrutinized each proof draft for classic failure modes — edge reuse, disconnected components, circular reasoning
  • 8-hour minimum budget: The prompt mandated at least 8 hours of continuous search before considering giving up — yet the system finished in under an hour
  • Assume completeness: The model was instructed from the start to assume a complete proof exists, a motivational trick to prevent early abandonment

Caveats and Controversy

Several important clarifications are needed:

  • No external verification: As of July 16, no mathematician outside OpenAI has reviewed the proof
  • Not machine-verified: Contrary to some reports, the proof was never converted into Lean or any other proof assistant format for machine checking
  • Classical tools used: The argument relies on existing graph theory results (Tutte's 8-flow theorem, reduction to cubic graphs) rather than novel mathematical machinery
  • Sensitive timing: The release came the day after Sol Ultra's public debut, raising questions about marketing considerations

Regardless of the final verdict, the 64-agent parallel reasoning + automatic attack verification paradigm itself represents a significant milestone for AI in mathematical research.

3. NVIDIA and LangChain Launch NemoClaw: Open Agent Matches Opus 4.8 at 1/10 Cost

NVIDIA and LangChain jointly released NemoClaw for LangChain Deep Agents, a reference blueprint stacking three open layers: NVIDIA's open-weight Nemotron 3 Ultra model, LangChain's Deep Agents harness for long-horizon task planning and tool execution, and NVIDIA's OpenShell sandbox runtime.

On LangChain's 135-task agent evaluation suite, Nemotron 3 Ultra wired into Deep Agents scored 0.86 at approximately $4.48 per run — compared to Claude Opus 4.8 at $43.48 per run, achieving near-parity at roughly one-tenth the cost.

Tuning the Harness, Not the Model

The most striking aspect: LangChain never fine-tuned Nemotron 3 Ultra. Instead, they analyzed agent execution traces to pinpoint failure patterns — wrong tool arguments, lost context in long tasks, misleading tool descriptions — then fixed the scaffolding: system prompts, tool descriptions, and middleware adjustments. This means the model weights stay frozen while performance improves through prompt engineering alone.

For enterprises, this translates to full ownership: every layer is open, the entire stack runs on self-managed hardware, and iterative improvement comes from trace analysis rather than expensive retraining runs.

Honest Positioning: Parity, Not Dominance

The "1/10 cost" headline primarily applies to the most expensive comparison — Opus 4.8. Against other open models like DeepSeek V4 and Kimi K2.6 ($10-15/run), Nemotron 3 Ultra is about 30% cheaper with slightly higher accuracy. Still, this marks the first time an fully open-source agent stack has reached practical parity with the best closed models on real business tasks — a milestone for organizations prioritizing data sovereignty and cost control.

EY has established an implementation practice around NemoClaw, and inference platforms including Baseten, Fireworks, and Nebius have onboarded support.
